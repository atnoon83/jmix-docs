== Основы

Компонент `integerField` предназначен для предоставления базовой функциональности ввода и отображения целых чисел.

Пример использования `integerField`:

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/integerfield/integerfield-view.xml[tags=basics]
----

image::visual-components/components/integer-field-basics.png[align="center", width="209"]

NOTE: `integerField` не поддерживает форматирование. Используйте xref:vc/components/textField.adoc[TypedTextField], когда вам нужно локализованное форматирование чисел или поддержка разных типов данных.

== Привязка данных

++++
<div class="jmix-ui-live-demo-container">
    <a href="https://demo.jmix.ru/ui-samples/sample/integer-field-dataaware" class="live-demo-btn" target="_blank">ДЕМОНСТРАЦИЯ</a>
</div>
++++

Привязка данных означает связывание визуального компонента с xref:data/data-containers.adoc[контейнером данных]. Изменения в визуальном компоненте или соответствующем контейнере данных могут вызывать обновления друг друга. Смотрите xref:flow-ui:data/data-examples.adoc[Использование компонентов данных] для получения дополнительной информации.

Следующий пример создает `integerField`, связанный с данными. Атрибут сущности должен быть типа `Integer`.

[source,xml,indent=0]
----
include::example$/onboarding/src/main/resources/com/company/onboarding/view/component/integerfield/integerfield-view.xml[tags=data;layout;data-binding]
----
<1> `InstanceContainer` для сущности `Step`.
<2> Встроенный план выборки экземпляра сущности, находящегося в контейнере.
<3> Привязка компонента к контейнеру данных и свойству. Атрибут `dataContainer` содержит ссылку на контейнер данных `stepDc`, а атрибут `property` ссылается на атрибут сущности `duration`.

[[about-step]]
== Шаг

++++
<div class="jmix-ui-live-demo-container">
    <a href="https://vaadin.com/docs/latest/components/number-field#step" class="vaadin-docs-btn" target="_blank">Vaadin</a>
</div>
++++

Атрибут `step` используется для управления значением увеличения или уменьшения при взаимодействии пользователя с полем с помощью стрелок вверх/вниз или кнопок шага. Шаг определяет, на сколько изменяется значение при каждом взаимодействии. Он помогает вам задать уровень точности для поля `integerField`.

NOTE: Атрибут `step` обычно используется в сочетании с атрибутами <<min,min>> и <<max,max>> для определения допустимого диапазона для поля `integerField`.

Атрибут `stepButtonsVisible` управляет тем, отображаются ли кнопки для увеличения/уменьшения значения или скрыты.

== Атрибуты

В Jmix есть множество xref:vc/common-attributes.adoc[общих атрибутов], которые выполняют одну и ту же функцию для всех компонентов.

Ниже приведены атрибуты, специфичные для `integerField`:

[cols="1h,5,1", grid=rows, frame=none]
|===
|*Название*
|*Описание*
|*Значение по умолчанию*
|[[max]]max
|Указывает максимальное значение поля. Ввод значения, которое больше `max`, делает поле недействительным.
|
|[[min]]min
|Указывает минимальное значение поля. Ввод значения, которое меньше `min`, делает поле недействительным.
|
|[[step]]step
|Задает допустимые числовые интервалы поля. Атрибут определяет, насколько увеличится/уменьшится значение при нажатии на кнопки шага. Смотрите <<about-step, Шаг>>.
|`1`
|[[stepButtonsVisible]]stepButtonsVisible
|Устанавливает видимость кнопок для увеличения/уменьшения значения в соответствии с заданным по умолчанию или указанным шагом. Смотрите <<about-step, Шаг>>.
|`false`
|===
